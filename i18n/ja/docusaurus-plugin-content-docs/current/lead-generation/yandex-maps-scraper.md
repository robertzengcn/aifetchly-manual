# Yandex Maps スクレイパー

aiFetchlyのYandex Mapsスクレイパーを使用すると、キーワードと場所を指定してYandex Mapsでローカルビジネスを検索できます。ロシアおよびCIS市場からのビジネスデータ収集に最適なツールであり、言語と地域のカスタマイズをサポートしています。

## 概要

Yandex Mapsスクレイパーは、包括的なローカルビジネスデータ抽出を提供します：

1. **キーワードを入力**（例：「歯医者」「レストラン」）
2. **場所を入力**（例：「モスクワ」「サンクトペテルブルク」「ロシア」）
3. **オプションを設定**（言語、地域、最大結果数など）
4. **検索を開始**してリアルタイムの進捗を確認
5. **結果をエクスポート**（CSV、JSON、またはクリップボードにコピー）

:::tip 最適な用途

Yandex Mapsスクレイパーは、ロシア市場およびCIS諸国をターゲットとするビジネスに不可欠です。Yandex Mapsは、ロシア、カザフスタン、ベラルーシ、トルコなど、Yandexが展開する地域のローカルビジネスを最も網羅的にカバーしています。

:::

## Yandex Mapsスクレイパーへのアクセス

1. 左側のナビゲーションメニューで**Yandex Maps**をクリックします
2. スクレイパーが**検索**と**履歴**の2つのタブで開きます

## 検索の実行

### ステップ1：検索条件の入力

#### ビジネスキーワード（必須）

検索するビジネスタイプまたは名前を入力します：

- **例**：`dentist`、`restaurant`、`plumber`、`marketing agency`
- **ヒント**：現地の言語でキーワードを入力すると最良の結果が得られます（例：ロシアの場所にはロシア語のキーワードを使用）

#### 場所（必須）

検索する都市または地域を入力します：

- **例**：`Moscow`、`Saint Petersburg`、`Russia`、`Almaty, Kazakhstan`
- **ヒント**：より具体的な場所を指定すると、より関連性の高い結果が得られます

### ステップ2：検索オプションの設定

#### 最大結果数

- **範囲**：1〜50件
- **デフォルト**：20件
- スライダーを調整して、抽出するビジネスの数を制御します

:::info 結果数の上限

信頼性の高い抽出を確保するため、最大結果数には安全な上限が設定されています。スライダーの範囲1〜50はユーザーが設定可能な範囲です。

:::

#### ウェブサイトを含む

- **有効**（デフォルト）：ビジネスのウェブサイトURLの抽出を試みます
- **無効**：ウェブサイトの抽出をスキップし、より高速な結果を得られます

#### レビューを含む

- **無効**（デフォルト）：基本のビジネスデータのみを返します
- **有効**：結果にレビューデータを含めます（処理時間が増加します）

#### ブラウザを表示

- **無効**（デフォルト）：ブラウザはヘッドレスモードで実行されます（より高速）
- **有効**：スクレイピング中にブラウザウィンドウが表示されます（デバッグに有用）

### ステップ3：言語と地域の設定（任意）

これらの設定はYandex Mapsに固有のもので、検索コンテキストのカスタマイズに役立ちます。

#### 言語

- Yandex MapsのUI言語コードを設定します
- **例**：`ru`（ロシア語）、`en`（英語）、`tr`（トルコ語）
- デフォルトの言語を使用する場合は空のままにします
- **ヒント**：その地域での検索結果を向上させるために、現地の言語を使用してください

#### 地域

- 検索コンテキストの地域コードを設定します
- **例**：`ru`（ロシア）、`kz`（カザフスタン）、`by`（ベラルーシ）
- デフォルトの地域を使用する場合は空のままにします
- **ヒント**：正しい地域を設定すると、結果の精度が向上します

### ステップ4：アカウントとプロキシの設定（任意）

#### Yandexアカウント

Yandexアカウントを選択すると、そのCookieを使用して認証済みスクレイピングが行われます：

- **メリット**：
  - 成功率の向上
  - より詳細なビジネス情報へのアクセス
  - ブロックされるリスクの軽減
- ドロップダウンにはYandexアカウントのみが表示されます

:::tip アカウントの追加

Yandexアカウントを追加するには、**設定**に移動してソーシャルアカウントを設定します。詳細は[システム設定](../settings/system-settings)を参照してください。

:::

#### プロキシ

検索中にローテーションするプロキシを1つ以上選択します：

- **メリット**：
  - 複数のIPアドレスにリクエストを分散
  - 検出リスクの軽減
  - 大規模スクレイピングに不可欠
- 複数のプロキシを選択すると、リクエストごとに自動的にローテーションされます

:::warning プロキシの推奨事項

Yandex Mapsのスクレイピングでは、ターゲット地域にあるプロキシ（例：モスクワの検索にはロシアのプロキシ）を使用すると、成功率が大幅に向上します。

:::

### ステップ5：検索の開始

1. **検索開始**をクリックして開始します
2. 円形の進捗インジケーターと進捗バーが表示されます
3. リアルタイムのステータステキストで現在のスクレイピング段階が表示されます
4. カウンターで進捗が表示されます（例：「5 / 20 ビジネス」）
5. 最大**3つの同時検索**を並行して実行できます

:::info 同時検索

Yandex Maps検索は最大3つまで同時に実行できます。4つ目を開始しようとすると、いずれかの完了を待つように促されます。

:::

### 検索のキャンセル

検索に時間がかかりすぎる場合や停止したい場合：

- 検索実行中に表示される**キャンセル**ボタンをクリックします
- キャンセル前に収集された部分的な結果は保持されます

## 結果の表示

検索が完了すると、結果がデータテーブルに表示されます。

### 結果テーブル

| 列 | 説明 |
|--------|-------------|
| **名前** | ビジネス名（Yandex Mapsへのリンクが利用可能な場合、クリック可能） |
| **カテゴリ** | ビジネスカテゴリ（例：「レストラン」「歯医者」） |
| **評価** | 星評価（星アイコン付き） |
| **レビュー** | レビュー数 |
| **住所** | 完全なビジネス住所 |
| **電話** | 電話番号 |
| **ウェブサイト** | ビジネスウェブサイトへのクリック可能なリンク（省略表示） |

:::tip ビジネス名をクリック

Yandex Mapsリンクのあるビジネス名はクリック可能で、クリックすると新しいタブでYandex Mapsのビジネスリスティングが開きます。

:::

### 検索結果ヘッダー

ヘッダーには以下が表示されます：

- 見つかったビジネスの総数
- 使用された検索キーワード
- 検索された場所

### 結果なし

ビジネスが見つからない場合、検索条件に対する結果がないことを示すメッセージが表示されます。以下をお試しください：

- より広範なキーワードを使用する
- 現地の言語でキーワードを入力する
- 検索場所を拡大する
- 最大結果数の設定を増やす

## 結果のエクスポート

### すべてコピー

1. 結果ヘッダーの**すべてコピー**をクリックします
2. すべての結果がJSONとしてクリップボードにコピーされます
3. 任意のテキストエディタまたはスプレッドシートに貼り付けます

### CSVでエクスポート

1. 結果ヘッダーの**CSVエクスポート**をクリックします
2. CSVファイルが自動的にダウンロードされます
3. ファイル名にはキーワードと場所が含まれます（例：`yandex-maps-dentist-Moscow.csv`）

**CSVの列：**
名前、カテゴリ、評価、レビュー数、住所、電話、ウェブサイト、営業時間、Maps URL

### JSONでエクスポート

1. 結果ヘッダーの**JSONエクスポート**をクリックします
2. 完全な結果データを含むJSONファイルがダウンロードされます
3. ファイル名はCSVエクスポートと同じ規則に従います

## 検索履歴

**履歴**タブには、過去のすべてのYandex Maps検索が保存されます。

### 履歴の表示

1. **履歴**タブをクリックします
2. 過去のすべての検索を示すテーブルが表示されます

| 列 | 説明 |
|--------|-------------|
| **クエリ** | 使用された検索キーワード |
| **場所** | 検索された場所 |
| **結果** | 見つかったビジネスの総数 |
| **日付** | 検索が実行された日時 |
| **アクション** | 表示または削除 |

### 履歴のアクション

- **表示**（目のアイコン）：履歴結果を検索タブに読み込みます
- **削除**（ゴミ箱アイコン）：検索記録を完全に削除します

### 履歴の更新

- **更新**ボタンをクリックして履歴リストを再読み込みします
- 履歴タブをクリックすると履歴が自動的に読み込まれます

## ベストプラクティス

### 1. キーワード戦略

- **現地の言語を使用**：ロシアの場所にはロシア語のキーワード
  - ✅ `стоматолог`（ロシア語で歯医者）でモスクワを検索
  - ✅ `dentist`で英語圏のエリアを検索
- **具体的にする**：正確なビジネスタイプを指定
- **バリエーションを試す**：包括的なカバーのため、複数のキーワードで検索

### 2. 場所のターゲティング

- **具体的な場所を使用**すると良い結果が得られます：
  - ✅ `Moscow`
  - ✅ `Saint Petersburg`
  - ✅ `Kazan, Russia`
  - ❌ `Russia`（範囲が広すぎる）
- **複数のエリアを検索**してより広い地域をカバー

### 3. 言語と地域

- ロシア語圏の地域では**言語を`ru`に設定**
- 最良の結果を得るために**地域をターゲット国に合わせて設定**
- 不明な場合は**空のままにする** — Yandexが場所に基づいてデフォルトを使用します

### 4. プロキシの使用

- **現地のプロキシを使用**（ロシアの検索にはロシアのプロキシ）
- **単一検索**：プロキシ不要
- **複数検索**：1〜2つのプロキシを使用
- **大規模スクレイピング**：3つ以上のプロキシをローテーションで使用

### 5. Yandexアカウントの使用

- すべてのYandex Mapsスクレイピングに**推奨**
- ビジネスが密集しているロシアの都市のスクレイピングには**必須**
- Yandexを積極的に使用しているアカウントは、より良い結果をもたらします

## 比較：Google Maps vs. Yandex Maps

| 機能 | Google Mapsスクレイパー | Yandex Mapsスクレイパー |
|---------|-------------------|-------------------|
| **最適な用途** | グローバル市場、欧米諸国 | ロシア、CIS、トルコ |
| **言語サポート** | 多言語（自動） | 言語/地域を設定可能 |
| **ビジネスカバレッジ** | グローバルで最高 | ロシア/CISで最高 |
| **アカウントタイプ** | Googleアカウント | Yandexアカウント |
| **エクスポート形式** | CSV、JSON | CSV、JSON、すべてコピー |
| **進捗トラッキング** | 進捗バー | 円形進捗 + ステータステキスト |

:::tip 両方のスクレイパーを使用する

地域を包括的にカバーするには、Google MapsとYandex Mapsの両方で検索を実行し、結果をクロスリファレンスして重複を排除してください。

:::

## トラブルシューティング

### 検索失敗

**考えられる原因：**
- ネットワーク接続の問題
- Yandex Mapsがリクエストをブロックした
- 無効なキーワードまたは場所

**解決策：**
1. インターネット接続を確認する
2. 別のキーワードまたは場所を試す
3. 認証アクセスのためにYandexアカウントを使用する
4. プロキシを有効にする — できればターゲット地域のIPを使用
5. 最大結果数の設定を減らす
6. 正しい言語と地域コードを設定してみる

### 結果が見つからない

**考えられる原因：**
- キーワードがそのエリアのビジネスと一致しない
- 場所が具体的すぎる、または曖昧すぎる
- 言語または地域の設定が間違っている

**解決策：**
1. より広範なキーワード、または現地の言語のキーワードを試す
2. より大きな都市や広範なエリアを使用する
3. Yandex Mapsでそのエリアにそのビジネスタイプが存在することを確認する
4. 言語と地域の設定を確認する

### 部分的な結果

**考えられる原因：**
- 完了前に検索がキャンセルされた
- 一部のビジネスリスティングに必要なデータが不足していた
- スクレイピング中にレート制限が発生した

**解決策：**
1. 検索が完全に完了するまで待つ
2. 残りのエリアについて別の検索を実行する
3. レート制限を回避するためにプロキシを使用する

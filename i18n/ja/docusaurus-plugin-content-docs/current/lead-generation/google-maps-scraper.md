# Google Maps スクレイパー

aiFetchlyのGoogle Mapsスクレイパーを使用すると、キーワードと場所を指定してGoogle Mapsでローカルビジネスを検索できます。ビジネス名、カテゴリ、評価、レビュー、住所、電話番号、ウェブサイトURLなど、Google Mapsの検索結果から包括的なビジネスデータを収集できます。

## 概要

Google Mapsスクレイパーは、ローカルビジネスデータを抽出するための合理化されたインターフェースを提供します：

1. **キーワードを入力**（例：「歯医者」「イタリアンレストラン」）
2. **場所を入力**（例：「ニューヨーク」「ロンドン」「90210」）
3. **オプションを設定**（最大結果数、ウェブサイト含む、レビューなど）
4. **検索を開始**してリアルタイムの進捗を確認
5. **結果をエクスポート**（CSVまたはJSON）

:::tip 最適な用途

Google Mapsスクレイパーは、確認済みの住所、電話番号、評価、営業時間などの構造化データを含むローカルビジネスの検索に最適です。一般的なウェブ検索結果よりも信頼性の高い情報が得られます。

:::

## Google Mapsスクレイパーへのアクセス

1. 左側のナビゲーションメニューで**Google Maps**をクリックします
2. スクレイパーが**検索**と**履歴**の2つのタブで開きます

## 検索の実行

### ステップ1：検索条件の入力

#### ビジネスキーワード（必須）

検索するビジネスタイプまたは名前を入力します：

- **例**：`dentist`、`Italian restaurant`、`plumber`、`marketing agency`
- **ヒント**：具体的なビジネスタイプを指定すると、より的確な結果が得られます

#### 場所（必須）

都市、住所、または郵便番号を入力します：

- **例**：`New York`、`London`、`90210`、`Paris, France`
- **ヒント**：より具体的な場所を指定すると、より関連性の高い結果が得られます

### ステップ2：検索オプションの設定

#### 最大結果数

- **範囲**：1〜50件
- **デフォルト**：20件
- スライダーを調整して、抽出するビジネスの数を制御します

#### ウェブサイトを含む

- **有効**（デフォルト）：ビジネスのウェブサイトURLの抽出を試みます
- **無効**：ウェブサイトの抽出をスキップし、より高速な結果を得られます

#### レビューを含む

- **無効**（デフォルト）：基本のビジネスデータのみを返します
- **有効**：結果にレビューデータを含めます（処理時間が増加します）

#### ブラウザを表示

- **無効**（デフォルト）：ブラウザはヘッドレスモードで実行されます（より高速）
- **有効**：スクレイピング中にブラウザウィンドウが表示されます（デバッグに有用）

:::warning ブラウザの表示

このオプションを有効にすると、検索中にブラウザウィンドウが画面に表示されます。これはデバッグ専用であり、スクレイピング処理が遅くなる可能性があります。

:::

### ステップ3：アカウントとプロキシの設定（任意）

#### Googleアカウント

Googleアカウントを選択すると、そのCookieを使用して認証済みスクレイピングが行われます：

- **メリット**：
  - 成功率の向上
  - より詳細なビジネス情報へのアクセス
  - ブロックされるリスクの軽減
- ドロップダウンにはGoogleアカウントのみが表示されます

:::tip アカウントの追加

Googleアカウントを追加するには、**設定**に移動してソーシャルアカウントを設定します。詳細は[システム設定](../settings/system-settings)を参照してください。

:::

#### プロキシ

検索中にローテーションするプロキシを1つ以上選択します：

- **メリット**：
  - 複数のIPアドレスにリクエストを分散
  - 検出リスクの軽減
  - 大規模スクレイピングに不可欠
- 複数のプロキシを選択すると、リクエストごとに自動的にローテーションされます

### ステップ4：検索の開始

1. **検索開始**をクリックして開始します
2. 進捗バーが表示され、リアルタイムのスクレイピング状況が示されます
3. 検索は非同期で実行されるため、他の機能を引き続き使用できます
4. 最大**3つの同時検索**を並行して実行できます

:::info 同時検索

Google Maps検索は最大3つまで同時に実行できます。4つ目を開始しようとすると、いずれかの完了を待つように促されます。

:::

### 検索のキャンセル

検索に時間がかかりすぎる場合や停止したい場合：

- 検索実行中に表示される**キャンセル**ボタンをクリックします
- キャンセル前に収集された部分的な結果は保持されます

## 結果の表示

検索が完了すると、結果がデータテーブルに表示されます。

### 結果テーブル

| 列 | 説明 |
|--------|-------------|
| **名前** | ビジネス名（Google Mapsへのリンクが利用可能な場合、クリック可能） |
| **カテゴリ** | ビジネスカテゴリ（例：「レストラン」「歯医者」） |
| **評価** | 星評価（星アイコン付き） |
| **レビュー** | レビュー数 |
| **住所** | 完全なビジネス住所 |
| **電話** | 電話番号 |
| **ウェブサイト** | ビジネスウェブサイトへのクリック可能なリンク（省略表示） |

:::tip ビジネス名をクリック

Google Mapsリンクのあるビジネス名はクリック可能で、クリックすると新しいタブでGoogle Mapsのビジネスリスティングが開きます。

:::

### 検索結果ヘッダー

ヘッダーには以下が表示されます：

- 見つかったビジネスの総数
- 使用された検索キーワード
- 検索された場所

### 結果なし

ビジネスが見つからない場合、検索条件に対する結果がないことを示すメッセージが表示されます。以下をお試しください：

- より広範なキーワードを使用する
- 検索場所を拡大する
- 最大結果数の設定を増やす

## 結果のエクスポート

### CSVでエクスポート

1. 結果ヘッダーの**CSVエクスポート**をクリックします
2. CSVファイルが自動的にダウンロードされます
3. ファイル名にはキーワードと場所が含まれます（例：`google-maps-dentist-NewYork.csv`）

**CSVの列：**
名前、カテゴリ、評価、レビュー数、住所、電話、ウェブサイト、営業時間、Maps URL

### JSONでエクスポート

1. 結果ヘッダーの**JSONエクスポート**をクリックします
2. 完全な結果データを含むJSONファイルがダウンロードされます
3. ファイル名はCSVエクスポートと同じ規則に従います

## 検索履歴

**履歴**タブには、過去のすべてのGoogle Maps検索が保存されます。

### 履歴の表示

1. **履歴**タブをクリックします
2. 過去のすべての検索を示すテーブルが表示されます

| 列 | 説明 |
|--------|-------------|
| **クエリ** | 使用された検索キーワード |
| **場所** | 検索された場所 |
| **結果** | 見つかったビジネスの総数 |
| **日付** | 検索が実行された日時 |
| **アクション** | 表示または削除 |

### 履歴のアクション

- **表示**（目のアイコン）：履歴結果を検索タブに読み込みます
- **削除**（ゴミ箱アイコン）：検索記録を完全に削除します

### 履歴の更新

- **更新**ボタンをクリックして履歴リストを再読み込みします
- 履歴タブをクリックすると履歴が自動的に読み込まれます

## ベストプラクティス

### 1. キーワード戦略

- **具体的にする**：正確なビジネスタイプを指定
  - ✅ `Italian restaurant`
  - ❌ `food`
- **バリエーションを試す**：包括的なカバーのため、複数のキーワードで検索
  - `dentist` と `dental clinic`
  - `plumber` と `plumbing service`

### 2. 場所のターゲティング

- **具体的な場所を使用**すると良い結果が得られます：
  - ✅ `Manhattan, New York`
  - ✅ `90210`
  - ❌ `USA`
- **複数のエリアを検索**してより広い地域をカバー：
  - 各地域や地区ごとに個別の検索を実行

### 3. 最大結果数の設定

- **クイックテスト**：5〜10件に設定して検索を確認
- **標準検索**：20件（デフォルト）
- **包括的**：50件で完全なカバー

### 4. プロキシの使用

- **単一検索**：プロキシ不要
- **連続した複数検索**：1〜2つのプロキシを使用
- **大規模スクレイピング**：3つ以上のプロキシをローテーションで使用

### 5. Googleアカウントの使用

- 30件以上の結果を返す検索には**推奨**
- ビジネスが密集している人気エリアのスクレイピングには**必須**
- Google Mapsを積極的に使用しているアカウントは、より良い結果をもたらします

## トラブルシューティング

### 検索失敗

**考えられる原因：**
- ネットワーク接続の問題
- Google Mapsがリクエストをブロックした
- 無効なキーワードまたは場所

**解決策：**
1. インターネット接続を確認する
2. 別のキーワードまたは場所を試す
3. 認証アクセスのためにGoogleアカウントを使用する
4. IPローテーションのためにプロキシを有効にする
5. 最大結果数の設定を減らす

### 結果が見つからない

**考えられる原因：**
- キーワードがそのエリアのビジネスと一致しない
- 場所が具体的すぎる、または曖昧すぎる
- Google Mapsが空の結果を返した

**解決策：**
1. より広範なキーワードを試す（例：`Italian restaurant`の代わりに`restaurant`）
2. より大きな都市や広範なエリアを使用する
3. Google Mapsでそのエリアにそのビジネスタイプが存在することを手動で確認する

### 部分的な結果

**考えられる原因：**
- 完了前に検索がキャンセルされた
- 一部のビジネスリスティングに必要なデータが不足していた
- スクレイピング中にレート制限が発生した

**解決策：**
1. 検索が完全に完了するまで待つ
2. 残りのエリアについて別の検索を実行する
3. レート制限を回避するためにプロキシを使用する

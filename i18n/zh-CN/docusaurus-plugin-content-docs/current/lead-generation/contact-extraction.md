# Contact Profile Insights

aiFetchly 的 Contact Profile Insights 可从网站批量构建结构化联系人档案。您可以从单个 URL 或现有搜索结果收集公开邮箱，并可选择使用 AI 为每个档案补充电话号码、地址和社交链接。

## 了解 Contact Profile Insights

Contact Profile Insights 的工作原理：

1. **访问您提供的每个 URL**
2. **扫描页面内容**查找邮件模式
3. **跟踪内部链接**（可选）
4. **选择适合 AI Enrichment 的候选页面**
5. **将发现的邮箱和补充字段**汇总为结构化列表
6. **自动去重**结果

:::info 使用场景

Contact Profile Insights 适用于：
- 从搜索结果中收集邮件地址
- 从行业目录建立联系人列表
- 从会员名录中获取联系信息
- 从资源页面提取邮件地址
- 使用电话号码、地址和社交资料补充联系人档案

:::

## 创建联系人档案任务

### 第 1 步：进入 Contact Profile Insights

1. 在左侧导航菜单中点击 **Contact Profile Insights**
2. 您将看到联系人档案任务列表
3. 点击 **Create New Task** 按钮

### 第 2 步：选择 URL 输入方式

选择提供 URL 进行提取的方式：

#### 方式 1：手动 URL 输入

**最适合**：自定义 URL 列表、特定网站

1. 从来源下拉菜单中选择 **Manual Input**
2. 在文本区域中输入您的 URL
3. **格式**：每行一个 URL

**示例：**
```
https://example.com
https://www.business-site.com/contact
https://another-site.com/about-us
```

**验证规则：**
- URL 必须以 `http://` 或 `https://` 开头
- 无效 URL 将自动标记
- 每个任务最多 10,000 个 URL

#### 方式 2：从搜索结果导入

**最适合**：利用现有搜索活动

1. 从来源下拉菜单中选择 **Search Results**
2. 显示您已完成搜索任务的表格
3. 选择包含您要提取 URL 的搜索任务
4. 点击 **Confirm**

**优势：**
- 与搜索功能无缝集成
- 无需手动输入 URL
- 使用之前已抓取的 URL

### 第 3 步：配置档案设置

#### 页面深度

- **默认值**：每个 URL 10 页
- **建议范围**：1-1000 页
- **用途**：对每个网站爬取多少页

**指导建议：**
- **小型网站**：5-10 页
- **中型网站**：10-50 页
- **大型网站**：50-100 页
- **超大型网站**：100+ 页（谨慎使用）

:::warning 页面数与时间

更高的页面深度 = 更长的提取时间。从保守设置开始，然后逐步增加。

:::

#### 并发数

- **默认值**：1 个并发进程
- **用途**：同时处理多少个 URL

**建议：**
- **不使用代理**：1-3 个并发
- **使用代理**：3-10 个并发
- **从低开始**逐步增加

#### 最大页数

- **默认值**：100 页
- **范围**：0-1000 页
- **用途**：处理页数的绝对上限

**使用场景**：防止在超大型网站上失控提取。

#### 处理超时

- **默认值**：10 分钟
- **范围**：1-20 分钟
- **用途**：每个 URL 超时前的最大处理时间

**调整场景：**
- 网站加载缓慢 → 增加超时时间
- 希望更快失败 → 减少超时时间

### 第 4 步：显示选项

#### 在浏览器中显示

- **否**（默认）：提取在后台运行（更快）
- **是**：浏览器窗口可见（调试模式）

**建议**：生产任务保持为"否"。

### 第 5 步：AI Enrichment（可选）

当您的账户启用 AI 后，任务表单会显示 **Enable AI Enrichment**。

- **关闭**：仅提取邮箱。
- **开启**：在可找到相关信息时，使用 AI 为每条结果补充电话号码、地址和社交链接。

AI Enrichment 适合需要更完整联系人档案来做外联或线索评估的场景。它可能增加处理时间；如果页面内容不足或 AI 补充请求无法完成，部分结果可能显示 **Skipped** 或 **Failed**。

### 第 6 步：代理配置（可选）

为大规模提取添加代理：

1. 点击 **Choose Proxy**
2. 选择一个或多个代理
3. 确认选择
4. 已选代理会以 chip 形式显示在代理选择器中

:::tip 何时使用代理

以下情况使用代理：
- 从 50+ 个 URL 提取
- 运行多个并发进程
- 之前的任务被阻止
- 从相同域名重复提取

:::

### 第 7 步：创建或更新任务

点击 **Submit** 创建新的联系人档案任务。新任务会提交到后台进程，任务启动后应用会返回任务列表。

编辑现有任务时，按钮会变为 **Save**。保存会更新 URL 来源、页面深度、并发数、超时、代理、浏览器可见性和 AI Enrichment。

## 管理提取任务

### 查看任务列表

导航到 **Contact Profile Insights** 查看所有任务。

**任务列表列说明：**

| 列 | 描述 |
|--------|-------------|
| **ID** | 唯一任务标识符 |
| **Type** | 手动输入或搜索结果 |
| **Status** | 待处理、处理中、已完成、错误 |
| **Record Time** | 任务创建时间 |
| **Actions** | 查看、停止、启动/重启、编辑、删除、下载日志 |

### 任务状态

| 状态 | 描述 | 操作 |
|--------|-------------|--------|
| **Pending** | 任务已创建但未开始 | 编辑、删除 |
| **Processing** | 正在查找公开联系人渠道 | 停止任务或监控进度 |
| **Complete** | 成功完成 | 查看结果、编辑设置或重启 |
| **Error** | 失败并出现错误 | 下载日志、编辑、删除或重启 |

### 任务操作

- **查看结果**（文件夹图标）：查看已提取的邮件
- **停止**（停止图标）：停止正在处理的任务
- **启动/重启**（播放图标）：启动当前未处理的任务
- **编辑**（铅笔图标）：修改任务设置
- **删除**（垃圾桶图标）：删除待处理或错误任务
- **下载日志**（下载图标）：获取错误日志（仅限失败任务）

## 查看联系人档案结果

### 第 1 步：访问结果

1. 进入 **Contact Profile Insights** 任务列表
2. 找到已完成的任务
3. 点击 **View Results**

### 第 2 步：结果表格

结果表格显示：

| 列 | 描述 |
|--------|-------------|
| **ID** | 唯一结果标识符 |
| **URL** | 来源网站 |
| **Record Time** | 结果采集时间 |
| **Phone** | AI 补充的电话号码（如有） |
| **Address** | AI 补充的地址（如有） |
| **Social Links** | AI 补充的社交资料链接（如有） |
| **AI Status** | 补充状态：Not enriched、Processing、Completed、Failed 或 Skipped |

### 第 3 步：展开详情

点击某行可展开查看：
- 在该 URL 上找到的所有邮件
- 邮件列表可复制
- 已补充的电话、地址和社交链接（如有）

### 第 4 步：搜索和筛选

- **搜索**：按 URL 或邮件地址筛选
- **分页**：浏览大型结果集
- **Export**：将任务结果下载为 CSV 文件

## 导出联系人档案结果

### 导出为 CSV

1. 打开任务结果详情
2. 点击 **Export**
3. aiFetchly 会将任务结果导出为 CSV，并返回保存的文件路径

**CSV 格式：**
```csv
URL,Email,Phone,Address,Social Links,AI Status,Timestamp
https://example.com,contact@example.com,+1-555-0100,"123 Market St","https://linkedin.com/company/example",completed,2026-06-08 10:30:00
```

### 在邮件营销中使用

提取的邮件可直接用于邮件营销：

1. **查看**提取任务的结果
2. 导出任务结果，或在邮件活动的邮件来源选择器中选择该任务
3. 使用收集到的邮箱和补充字段进行审核和个性化

详细说明请参阅[批量邮件发送](./batch-email-sending)。

## 最佳实践

### 1. URL 来源策略

**高质量来源：**
- 行业目录
- 会员名录
- 协会网站
- 资源页面
- "联系我们"页面

**避免：**
- 社交媒体平台（很少有邮件地址）
- 新闻网站（转化率低）
- 超大型门户网站（质量低）

### 2. 页面深度设置

**保守模式**（注重质量）：
- 页面深度：5-10
- 并发数：1-3
- AI Enrichment：高价值目标建议开启
- 最适合：针对性列表、高价值联系人

**中等模式**（平衡）：
- 页面深度：10-50
- 并发数：3-5
- AI Enrichment：需要更高档案质量时开启
- 最适合：一般外联活动

**激进模式**（注重数量）：
- 页面深度：50-100+
- 并发数：5-10
- AI Enrichment：选择性使用以控制处理时间
- 最适合：市场研究、广泛覆盖

:::warning 质量与数量

激进设置可能提取更多邮件但质量较低。专注于相关来源以获得更好的营销活动效果。

:::

### 3. 代理使用

**小型任务**（< 100 个 URL）：
- 不需要代理
- 并发数：1-3

**中型任务**（100-1000 个 URL）：
- 使用 2-3 个代理
- 并发数：3-5

**大型任务**（1000+ 个 URL）：
- 使用 5+ 个代理
- 并发数：5-10
- 定期轮换代理

### 4. 去重

aiFetchly 自动在任务内去重邮件。额外去重方法：

- 将结果导出为 CSV
- 使用电子表格软件或脚本
- 与现有联系人列表比较
- 在营销活动前移除重复项

### 5. 邮件验证

提取的邮件可能并非总是有效的。建议：

- **手动审查**：抽查邮件格式
- **邮件验证工具**：使用第三方服务
- **测试营销活动**：先发送小批量
- **跟踪退信**：移除无法送达的邮件

## 与搜索结果的集成

最强大的工作流程是结合搜索和提取：

### 完整工作流程

1. **运行搜索任务**：
   - 搜索目标行业的企业
   - 使用 AI 关键词生成实现全面覆盖

2. **在 Contact Profile Insights 中打开**：
   - 从搜索结果创建联系人档案任务
   - 从发现的 URL 打开 Contact Profile Insights

3. **质量控制**：
   - 审查提取的邮件和补充的联系人字段
   - 按来源质量筛选
   - 移除重复项

4. **邮件营销活动**：
   - 导入到邮件营销
   - 创建个性化模板
   - 发起营销活动

## 高级技巧

### 技巧 1：深度提取

用于全面的邮件收集：

1. **设置页面深度**：50-100 页
2. **启用代理**：使用 3-5 个代理
3. **中等并发**：3-5
4. **监控进度**：定期检查结果
5. **提前终止**：如果质量下降，调整设置

### 技巧 2：基于模式的提取

针对特定类型的页面：

- **联系页面**：URL 包含 `/contact`
- **关于页面**：URL 包含 `/about`
- **团队页面**：URL 包含 `/team`
- **会员目录**：协会网站

### 技巧 3：竞争对手分析

从竞争对手网站提取邮件：

1. **确定**您所在领域的竞争对手
2. **提取他们的联系邮件**
3. **分析他们的合作伙伴**（链接页面）
4. **建立合作伙伴外联列表**

## 故障排除

### 任务状态："Error"

**可能原因：**
- 无效 URL
- 网络连接问题
- 所有代理均失败
- 网站阻止访问

**解决方案：**
1. 验证 URL 格式（http:// 或 https://）
2. 检查网络连接
3. 测试代理健康状况
4. 减少并发数
5. 增加超时设置

### 未提取到邮件

**可能原因：**
- 网站没有公开可见的邮件地址
- 邮件在图片/JavaScript 中（无法提取）
- 网站使用联系表单而非邮件
- 页面深度设置过低

**解决方案：**
1. 增加页面深度设置
2. 手动验证网站是否有邮件地址
3. 尝试不同的 URL 来源
4. 检查网站是否使用联系表单

### AI Enrichment 不可用

**可能原因：**
- 您的账户未启用 AI
- 应用无法加载当前 AI 账户状态

**解决方案：**
1. 确认您的账户或工作区已启用 AI 功能
2. 启用 AI 后重新打开任务表单
3. 如果只需要邮箱地址，可以不启用 AI Enrichment 运行任务

### AI Status 显示 Failed 或 Skipped

**可能原因：**
- 页面没有足够有用的联系人上下文
- 网站阻止访问联系页面
- 该结果的 AI Enrichment 超时或失败

**解决方案：**
1. 展开该行，查看仍然成功提取的邮箱
2. 如果联系页面层级较深，提高页面深度
3. 如果页面被阻止，降低并发数或使用代理
4. 调整设置后重启任务

### 处理速度慢

**可能原因：**
- 页面深度过高
- 并发进程过多
- 网站响应缓慢
- 网络延迟

**解决方案：**
1. 减少页面深度
2. 降低并发数
3. 增加超时时间
4. 使用更快的代理

### 邮件重复

**可能原因：**
- 同一邮件出现在多个页面上
- 来自同一域名的多个 URL

**解决方案：**
1. aiFetchly 自动在任务内去重
2. 导出并在任务间去重
3. 使用电子表格软件或脚本
4. 使用邮件验证工具

### 被网站阻止

**可能原因：**
- 并发请求过多
- 未轮换代理
- 设置过于激进

**解决方案：**
1. 将并发数降至 1-3
2. 使用多个代理
3. 增加请求之间的延迟
4. 遵守网站速率限制

## 法律和道德注意事项

### 合规性

提取邮件时，请考虑：

- **GDPR**（欧洲）：对邮件收集的严格法规
- **CAN-SPAM**（美国）：商业邮件要求
- **CASL**（加拿大）：电子消息的同意要求

:::warning 法律声明

始终确保您拥有合法权利提取和联系这些邮件地址。请咨询法律顾问了解适用法律的指导。

:::

### 最佳实践

- **仅使用公开来源**：从公开可用的信息中提取
- **相关上下文**：从与您的产品/服务相关的企业/联系人处提取
- **尊重 Robots.txt**：遵守网站排除标准
- **提供退订选项**：在邮件中包含退订选项
- **价值主张**：在外联中提供有价值的内容

## 与邮件营销的集成

提取邮件后：

1. **审查结果**：质量检查您提取的邮件
2. **审查补充信息**：启用 AI Enrichment 时，检查电话、地址、社交链接和 AI Status
3. **导出或导入**：在营销活动中选择该任务或导出为 CSV
4. **选择模板**：选择或创建邮件模板
5. **个性化**：使用 AI 邮件撰写器生成个性化内容
6. **发起营销活动**：发送针对性外联

创建营销活动的详细说明请参阅[批量邮件发送](./batch-email-sending)。

# 任务调度

aiFetchly 的任务调度器允许您通过在指定时间自动运行任务或响应其他任务来自动化您的营销工作流程。设置定期搜索、自动电子邮件营销活动和复杂的多步骤工作流程。

## 了解调度

任务调度器支持三种类型的任务触发器：

| 触发器类型 | 描述 | 适用场景 |
|-------------|-------------|----------|
| **Cron** | 使用 cron 表达式进行基于时间的调度 | 循环任务、每日/每周/每月任务 |
| **依赖** | 由另一个任务的完成触发 | 多步骤工作流程、任务链 |
| **手动** | 仅在手动触发时执行 | 按需任务、测试 |

:::info 全自动运行

从潜在客户获取到电子邮件营销活动，调度自动化可以节省时间并确保您的营销工作流程一致执行。

:::

## 创建计划任务

### 第 1 步：进入调度器

1. 点击左侧导航菜单中的 **Schedule**
2. 点击 **New Schedule** 按钮

### 第 2 步：基本信息

输入以下必填信息：

#### 调度名称

- **用途**：在列表中标识该调度
- **示例**："每日 Google 搜索"、"每周电子邮件营销活动"
- **必填**：是

#### 任务类型

选择要调度的任务类型：

- **Search**：搜索引擎抓取任务
- **Email Extract**：电子邮件提取任务
- **Outreach Campaign**：电子邮件营销活动
- **Directory Assistant**：目录抓取任务
- **Google Maps**：Google 地图抓取任务
- **Yandex Maps**：Yandex 地图抓取任务
- **AI Message**：AI 消息任务，支持工具集成

#### 任务 ID

- **用途**：关联到特定的任务实例
- **选择**：从所选类型的现有任务中选择
- **必填**：是
- **注意**：对于 AI Message 任务，任务配置将在设置调度时内联创建（见下方 [AI Message 任务配置](#ai-message-任务配置)）

#### 描述

- **用途**：提供有关调度目的的上下文信息
- **示例**："每日搜索目标城市中的新营销机构"
- **选填**：是

### AI Message 任务配置

当您选择 **AI Message** 作为任务类型时，将显示额外的配置表单来定义 AI 任务。此部分不会出现在其他任务类型中。

#### 任务名称

- **用途**：标识 AI 消息任务
- **必填**：是
- **示例**："每日潜在客户分析"、"自动回复咨询"

#### AI 消息

- **用途**：发送给 AI 模型的提示/消息
- **必填**：是
- **示例**："分析今天的搜索结果，根据公司规模和行业识别出最有潜力的 10 个潜在客户"
- **支持**：包含详细说明的多行文本

#### 允许的工具

选择 AI 代理在执行期间可以使用的工具：

| 工具风险等级 | 颜色 | 描述 |
|----------------|-------|-------------|
| **低** | 绿色 | 安全的只读操作 |
| **中** | 黄色 | 影响适中的操作 |
| **高** | 红色 | 修改数据或执行重要操作 |
| **已阻止** | — | 不允许用于计划任务的工具 |

- **用途**：控制 AI 可以自主执行的操作
- **选择**：从可用的可调度工具中多选
- **默认**：未选择任何工具

#### 自动批准工具

- **用途**：允许 AI 无需手动批准即可执行工具调用
- **默认**：关闭（禁用）
- **警告**：启用此选项意味着 AI 可以自动运行工具。启用前请仔细审查允许的工具列表。

:::caution 自动批准安全

当启用自动批准时，AI 将在无需等待人工确认的情况下执行已批准的工具。仅在对经过充分测试的任务配置和精心策划的工具列表启用此功能。请务必设置适当的安全限制。

:::

#### 安全限制

配置限制以将 AI 任务保持在安全范围内：

| 设置 | 默认值 | 最小值 | 描述 |
|---------|---------|-----|-------------|
| **最大工具调用次数** | 10 | 1 | 每次执行的最大工具调用次数 |
| **最大运行时间** | 300,000 毫秒（5 分钟） | 1,000 毫秒 | 最大执行时间 |
| **最大继续调用次数** | 10 | 1 | 最大续接循环次数 |

这些限制可防止失控任务消耗过多资源或无限期执行。

#### Cron 调度（基于时间）

启用 **Cron** 并配置调度计划：

**预设选项：**
- 每分钟
- 每小时
- 每天（午夜）
- 每周（周日午夜）
- 每月（每月 1 日午夜）
- 每 15 分钟
- 每 30 分钟
- 每 2 小时
- 每 6 小时
- 每 12 小时
- 工作日上午 9 点
- 周末上午 10 点

**自定义 Cron 构建器：**

| 字段 | 选项 | 描述 |
|-------|---------|-------------|
| **分钟** | `*/5`、`*/15`、`*/30` 或特定分钟 | 每 5/15/30 分钟或特定时间 |
| **小时** | `*/2`、`*/6`、`*/12` 或特定小时 | 每 2/6/12 小时或特定时间 |
| **日** | `*/2` 或特定日期 | 每 2 天或特定日期 |
| **月** | `*/3`、`*/6` 或特定月份 | 每 3/6 个月或特定月份 |
| **星期** | `1-5`（工作日）、`0,6`（周末）或特定日期 | 工作日、周末或特定日期 |

**Cron 表达式格式：**
```
┌───────────── minute (0 - 59)
│ ┌───────────── hour (0 - 23)
│ │ ┌───────────── day of month (1 - 31)
│ │ │ ┌───────────── month (1 - 12)
│ │ │ │ ┌───────────── day of week (0 - 6) (Sunday to Saturday)
│ │ │ │ │
* * * * *
```

**示例：**
```
0 9 * * 1-5      # 工作日上午 9:00
*/30 * * * *     # 每 30 分钟
0 0 * * 1        # 每周一午夜
0 9,15 * * *     # 每天上午 9:00 和下午 3:00
0 0 1 * *        # 每月 1 日午夜
```

**下次运行时间：**
- 系统自动计算并显示下次执行时间
- 有助于验证您的 cron 表达式

#### 依赖调度（基于任务）

启用 **Dependency** 以在另一个任务完成时触发此任务。

**配置：**

1. **父调度**：选择将触发此任务的调度
2. **依赖条件**：
   - **On Success**：仅在父任务成功时运行
   - **On Completion**：在父任务完成后运行（无论成功或失败）
   - **On Failure**：仅在父任务失败时运行

3. **延迟（分钟）**：父任务完成后等待多长时间再运行此任务
   - `0` 分钟：立即运行
   - `5` 分钟：等待 5 分钟后开始
   - `60` 分钟：等待 1 小时后开始

**使用场景：**
- **提取后发送邮件**：提取电子邮件，完成后发送营销活动
- **抓取后分析**：抓取数据，然后运行 AI 分析
- **多阶段营销活动**：初始联系 → 跟进 1 → 跟进 2

:::tip 依赖链

您可以创建多层依赖链：
- 任务 A（Cron）→ 任务 B（依赖）→ 任务 C（依赖）

这将创建强大的自动化工作流程。

:::

#### 手动执行

启用 **Manual** 以仅在触发时运行任务：

- 无自动调度
- 通过 "Run Now" 按钮按需执行
- 适用于测试或不频繁的任务

### 第 4 步：激活状态

切换调度的状态：
- **Active**：调度将根据触发器设置运行
- **Inactive**：调度已禁用，不会运行

### 第 5 步：保存调度

点击 **Save** 以创建调度。您可以：
- 稍后 **Edit**（编辑）调度
- 根据需要 **Enable/Disable**（启用/禁用）
- **Run Now** 立即测试

## 管理计划任务

### 查看调度列表

导航到 **Schedule** 查看所有计划任务。

**调度概览：**
- **状态卡片**：显示总数、活跃、非活跃和已暂停的调度
- **过滤器**：按名称、状态、任务类型、触发器类型搜索

### 调度表格

调度列表显示以下信息：

| 列 | 描述 |
|--------|-------------|
| **Name** | 带有任务类型指示器的调度名称 |
| **Status** | Active（绿色）、Inactive（灰色）、Paused（黄色） |
| **Trigger Type** | Cron、Dependency 或 Manual |
| **Schedule** | Cron 表达式或依赖描述 |
| **Next Run** | 下次执行时间及倒计时 |
| **Last Run** | 最近执行时间及已用时间 |
| **Executions** | 成功次数 / 失败次数 |
| **Actions** | 编辑、删除、暂停/恢复、立即运行 |

### 调度操作

| 操作 | 描述 | 可用条件 |
|--------|-------------|----------------|
| **Edit** | 修改调度配置 | 始终可用 |
| **Delete** | 删除调度 | 始终可用 |
| **Enable** | 激活非活跃的调度 | 非活跃调度 |
| **Disable** | 停用调度 | 活跃调度 |
| **Pause** | 暂时停止执行 | 活跃调度 |
| **Resume** | 恢复已暂停的调度 | 已暂停的调度 |
| **Run Now** | 立即执行 | 始终可用 |

## 监控执行

### 查看执行历史

1. 在调度上点击 **View Details**
2. 查看执行历史表格

**执行信息：**

| 列 | 描述 |
|--------|-------------|
| **Start Time** | 执行开始时间 |
| **End Time** | 执行完成时间（或显示 "Running..."） |
| **Duration** | 执行耗时（或实时计数器） |
| **Status** | Success（绿色）、Failed（红色）、Running（蓝色） |
| **Result** | 执行结果摘要 |
| **Error** | 错误信息（如果执行失败） |
| **Actions** | 查看详情、取消运行中的任务 |

### 执行统计

详情视图显示：

- **成功率**：成功执行的百分比
- **平均耗时**：典型执行时间
- **总执行次数**：总计数及明细
- **最近错误**：最近的失败信息
- **下次运行**：距离下次执行的倒计时

### 实时监控

对于运行中的任务：
- **实时耗时**：计数器显示已用时间
- **取消按钮**：如需停止执行
- **自动刷新**：状态每隔几秒更新

## 常见调度模式

### 模式 1：每日潜在客户获取

**调度**：每个工作日上午 9:00
```
Cron: 0 9 * * 1-5
Task: Market Insight Explorer
```

**使用场景**：每天早上为您的销售团队获取新的潜在客户。

### 模式 2：每小时监控

**调度**：工作时间内的每小时
```
Cron: 0 9-17 * * 1-5
Task: Search / Yellow Pages
```

**使用场景**：在工作时间内监控新的商家列表。

### 模式 3：每周营销活动

**调度**：每周一上午 10:00
```
Cron: 0 10 * * 1
Task: Outreach Campaign Sending
```

**使用场景**：每周电子邮件简报或推广活动。

### 模式 4：多步骤工作流程

**任务 A**（Cron）：每天上午 9:00 - 搜索潜在客户
**任务 B**（依赖）：A 成功后 - 提取电子邮件
**任务 C**（依赖）：B 成功后（延迟 30 分钟）- 发送电子邮件

**使用场景**：自动化的潜在客户获取和推广流程。

### 模式 5：维护任务

**调度**：每周日凌晨 3:00
```
Cron: 0 3 * * 0
Task: Data cleanup or backup
```

**使用场景**：低流量时段的日常维护。

### 模式 6：AI 驱动的潜在客户分析

**调度**：每个工作日上午 10:00
```
Cron: 0 10 * * 1-5
Task: AI Message
```

**AI 配置**：
- **消息**："审查昨天的搜索结果，分析潜在客户质量，并生成最有前景联系人的摘要报告"
- **允许的工具**：选择相关的数据分析工具
- **自动批准**：启用（已仔细审查工具列表）
- **最大运行时间**：300,000 毫秒（5 分钟）

**使用场景**：使用 AI 自动分析每日收集的潜在客户。

## 最佳实践

### 1. 调度设计

**避免重叠：**
- 确保任务在下一次计划运行之前完成
- 在设置频率时考虑平均执行时间
- 使用依赖来排序重叠的任务

**非高峰时段：**
- 在非工作时间调度资源密集型任务
- 避免与用户活动竞争
- 考虑全球运营的时区问题

**缓冲时间：**
- 在依赖任务之间添加缓冲
- 考虑可变的执行时间
- 防止级联延迟

### 2. 错误处理

**监控失败：**
- 定期检查执行历史
- 调查重复失败
- 根据需要调整调度或任务

**设置警报：**
- 每周审查调度
- 检查卡住或暂停的任务
- 验证依赖是否正常触发

**优雅降级：**
- 使用 "On Completion" 依赖，即使某个任务失败也能继续链路
- 为关键任务创建替代调度
- 记录升级处理程序

### 3. 资源管理

**并发任务：**
- 避免同时调度过多任务
- 考虑系统资源（CPU、内存、网络）
- 错开类似任务以防止冲突

**代理轮换：**
- 确保有足够的代理用于并发的计划任务
- 在代理池中分配负载
- 监控计划任务的代理健康状况

### 4. 测试

**测试调度：**
- 在正式调度前使用 "Run Now" 进行测试
- 先通过一次性执行进行验证
- 检查日志中的任何问题

**验证 Cron 表达式：**
- 使用 "Next Run Time" 预览进行验证
- 先用较短的间隔测试
- 确认时区设置正确

**测试依赖：**
- 验证父任务是否成功完成
- 测试延迟设置
- 确保链路按预期工作

### 5. 文档记录

**清晰命名调度：**
- 使用描述性名称，包含用途和频率
- 包含任务类型和目标
- 示例："每日 Google 搜索 - 营销机构"

**使用描述：**
- 记录目的和预期结果
- 注明依赖和关系
- 包含任何特殊注意事项

**标记任务用途：**
- 对相关调度进行标记或分类
- 按项目或营销活动分组
- 便于识别

## 故障排除

### 调度未运行

**可能原因：**
- 调度处于非活跃或已暂停状态
- Cron 表达式配置错误
- 调度器服务未运行
- 系统时间/时区问题

**解决方案：**
1. 验证调度状态为 "Active"
2. 检查 cron 表达式语法
3. 确认调度器服务正在运行
4. 验证系统时间和时区设置
5. 检查执行日志中的错误

### 任务执行过于频繁

**可能原因：**
- Cron 表达式不正确
- 同一任务有多个调度
- 对 cron 语法的误解

**解决方案：**
1. 仔细审查 cron 表达式
2. 检查是否有重复调度
3. 使用 "Next Run Time" 预览进行验证
4. 先用较长的间隔测试

### 依赖未触发

**可能原因：**
- 父任务未完成
- 依赖条件设置错误
- 延迟过长或过短

**解决方案：**
1. 检查父任务执行历史
2. 验证依赖条件是否符合预期行为
3. 调整延迟设置
4. 检查是否存在循环依赖

### 任务耗时过长

**可能原因：**
- 任务配置过于激进
- 系统资源不足
- 网络瓶颈

**解决方案：**
1. 减少任务范围（页面数、并发数等）
2. 在非高峰时段调度
3. 增加运行间隔
4. 检查系统性能

### AI Message 任务问题

**任务触及安全限制：**
- **可能原因**：达到最大工具调用次数、最大运行时间或最大续接调用次数
- **解决方案**：
  1. 审查 AI 消息提示的清晰度和具体性
  2. 如果任务确实需要更多资源，增加安全限制
  3. 缩小任务提示的范围
  4. 检查执行日志以了解触及了哪个限制

**AI 任务产生意外结果：**
- **可能原因**：提示不明确、选择了错误的工具或上下文不足
- **解决方案**：
  1. 使用更具体的说明优化 AI 消息
  2. 审查和调整允许的工具列表
  3. 在禁用自动批准的情况下测试，手动审查工具调用
  4. 在系统提示中添加更多上下文

**工具调用被阻止：**
- **可能原因**：工具不在允许列表中，或工具被标记为计划任务不可用
- **解决方案**：
  1. 将所需工具添加到允许的工具列表
  2. 验证工具可用于计划执行
  3. 检查工具风险等级，确认未被策略阻止

### 执行历史未显示

**可能原因：**
- 任务从未执行过
- 历史记录已清除
- 数据库问题

**解决方案：**
1. 手动运行任务进行测试
2. 检查任务是否曾经执行过
3. 验证数据库连接
4. 如需要，重启调度器服务

## 高级工作流程

### 工作流程 1：自动化潜在客户获取流程

**调度 1**：每日搜索
```
Cron: 0 9 * * 1-5 (Weekdays 9 AM)
Task: Google Search for "marketing agencies [city]"
```

**调度 2**：电子邮件提取（依赖）
```
Trigger: After Schedule 1 succeeds
Delay: 0 minutes
Task: Open in Contact Profile Insights from Schedule 1 results
```

**调度 3**：电子邮件营销活动（依赖）
```
Trigger: After Schedule 2 completes
Delay: 60 minutes (allow time for profile insights)
Task: Send welcome email campaign
```

**结果**：自动化的每日潜在客户获取和推广。

### 工作流程 2：每周维护

**调度 1**：数据库清理
```
Cron: 0 3 * * 0 (Sunday 3 AM)
Task: Remove old completed tasks
```

**调度 2**：代理健康检查（依赖）
```
Trigger: After Schedule 1 completes
Task: Test all proxies and remove failed
```

**调度 3**：报告生成（依赖）
```
Trigger: After Schedule 2 completes
Task: Generate weekly usage report
```

**结果**：自动化的每周维护和报告。

### 工作流程 3：多区域监控

**调度 1**：美国东部监控
```
Cron: 0 */2 * * * (Every 2 hours)
Task: Search US East keywords
```

# 系统设置

系统设置部分允许您配置 aiFetchly 的核心功能，包括用于 Web 自动化的浏览器路径、验证码求解、嵌入模型和系统偏好设置。正确的配置可确保所有功能的最佳性能。

## 访问系统设置

1. 点击左侧导航菜单中的 **设置**
2. 左侧将出现一个树形导航面板
3. 点击任意类别以展开其设置
4. 根据需要修改设置
5. 更改会自动保存

:::info 自动保存

大多数设置在修改时会自动保存。请注意保存指示器或确认消息。

:::

## 设置概览

### 2Captcha 服务

用于网页抓取的自动验证码求解。

| 设置 | 描述 |
|------|------|
| **Token** | 您的 2Captcha API 令牌 |
| **已启用** | 开启/关闭验证码求解 |

**设置步骤（可选）：**
1. 在 https://2captcha.com 注册
2. 向账户充值
3. 从控制面板获取 API 令牌
4. 输入令牌并启用服务

:::info 何时使用 2Captcha

在以下情况下有用：
- 大规模抓取
- 频繁遇到验证码
- 不想手动解决验证码

:::

### 嵌入配置

配置用于 RAG（检索增强生成）的默认嵌入模型。

| 设置 | 描述 |
|------|------|
| **默认模型** | 从可用的嵌入模型中选择 |

**选项：**
- 具有不同维度的各种嵌入模型
- 根据以下因素选择：
  - 语言支持
  - 性能要求
  - 资源限制

### 外部系统路径

配置用于本地浏览器集成的浏览器路径。

#### Chrome 路径

**用途**：Chrome 浏览器可执行文件的路径

**配置方法：**
1. 点击 **浏览** 或文件选择按钮
2. 导航到 Chrome 安装目录
3. 选择可执行文件

**默认路径（按操作系统）：**
- **Windows**：`C:\Program Files\Google\Chrome\Application\chrome.exe`
- **macOS**：`/Applications/Google Chrome.app/Contents/MacOS/Google Chrome`
- **Linux**：`/usr/bin/google-chrome` 或 `/usr/bin/chromium-browser`

#### Firefox 路径

**用途**：Firefox 浏览器可执行文件的路径

**配置方法：**
1. 点击 **浏览** 或文件选择按钮
2. 导航到 Firefox 安装目录
3. 选择可执行文件

**默认路径（按操作系统）：**
- **Windows**：`C:\Program Files\Mozilla Firefox\firefox.exe`
- **macOS**：`/Applications/Firefox.app/Contents/MacOS/firefox`
- **Linux**：`/usr/bin/firefox`

:::tip 浏览器路径要求

浏览器路径在以下情况下是必需的：
- 本地浏览器集成功能
- Yandex 抓取（需要浏览器）
- 某些反机器人检测场景

:::

### 用户偏好

配置您的 aiFetchly 使用体验。

#### 语言

**选项：**
- **English**：英文界面
- **中文**：简体中文界面

**更改方法：**
1. 从下拉菜单中选择首选语言
2. 界面将立即更新
3. 建议重启以完全切换语言

#### AI 网站分析业务信息

**用途**：为 AI 驱动的网站分析提供上下文

**格式**：JSON 配置

**示例：**
```json
{
  "industry": "Software",
  "company_size": "50-200 employees",
  "target_markets": ["B2B", "SaaS"],
  "keywords": ["marketing", "automation"]
}
```

**用法：**
- AI 在分析网站时使用此上下文
- 提高分析结果的相关性
- 自定义评分和分类

## MCP 工具管理

**MCP**（模型上下文协议）允许与外部工具和服务集成，通过网页搜索、数据库查询和自定义 API 调用等功能扩展 AI 营销助手。

有关配置和使用 MCP 服务器的完整文档，请参阅 [MCP 工具](../ai-outreach/mcp-tools)。

## 配置最佳实践

### 1. 浏览器配置

**Chrome 与 Firefox：**
- Chrome：兼容性更好，功能更多
- Firefox：开源，注重隐私

**建议：** 将 Chrome 配置为主要浏览器，Firefox 作为备用。

### 2. 验证码求解

**何时启用：**
- 大规模抓取操作（>1000 页）
- 频繁遇到验证码
- 不希望手动干预

**何时禁用：**
- 小规模抓取
- 为了节省成本（2captcha 会产生费用）
- 很少遇到验证码

### 3. 语言设置

**选择依据：**
- 您的母语
- 目标受众的语言
- 内容语言（用于知识库）

**注意：** 语言偏好仅影响用户界面。AI 可以处理多种语言，不受此设置影响。

### 4. MCP 工具

**适量添加：**
- 仅添加您会使用的工具
- 每个工具都会增加复杂性
- 逐一测试工具

**安全性：**
- 仅使用受信任的 MCP 服务器
- 妥善保管凭据
- 审查工具权限

## 故障排除

### 浏览器集成不工作

**可能的原因：**
- 浏览器路径不正确
- 浏览器未安装
- 权限问题

**解决方案：**
1. 验证浏览器已安装
2. 检查文件路径是否正确
3. 通过直接启动浏览器进行测试
4. 以管理员/sudo 权限运行 aiFetchly

### 验证码无法求解

**可能的原因：**
- 2Captcha 令牌无效
- 余额不足
- 服务已禁用

**解决方案：**
1. 验证 2Captcha 令牌是否正确
2. 检查账户余额
3. 确保 2Captcha 开关已启用
4. 先测试手动验证码

### 设置未保存

**可能的原因：**
- 数据库被锁定
- 权限不足
- 应用程序错误

**解决方案：**
1. 重启 aiFetchly
2. 以管理员/sudo 身份运行
3. 检查应用程序日志
4. 验证数据库不是只读的

### MCP 工具未显示

**可能的原因：**
- 服务器配置不正确
- 连接测试失败
- 工具已禁用

**解决方案：**
1. 验证服务器配置
2. 测试连接
3. 检查服务器是否正在运行
4. 启用各个工具

## 高级配置

### 多个嵌入模型

为不同目的配置不同的嵌入模型：

1. 导航到 **嵌入配置**
2. 添加多个模型
3. 设置默认模型
4. 按任务使用特定模型

### 特定环境设置

不同环境使用不同设置：

**开发环境：**
- 禁用验证码求解
- 启用调试日志

**生产环境：**
- 启用验证码求解
- 禁用调试日志

## 安全注意事项

### 浏览器路径

**安全性：**
- 仅使用受信任的浏览器安装
- 验证浏览器可执行文件是否合法
- 保持浏览器更新
- 对自定义浏览器构建保持谨慎

### MCP 工具

**安全性：**
- 仅连接到受信任的 MCP 服务器
- 仔细审查工具权限
- 只要可用就使用身份验证
- 监控工具使用情况
- 不需要时撤销访问权限
